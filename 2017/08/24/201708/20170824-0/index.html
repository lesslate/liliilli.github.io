어플리케이션 알림 설정확인 방법을 통해 알게된 것들

Hexo
2017-08-24
Android

하계인턴을 가기 전까지는, 어플리케이션 자체에서 푸시 알람을 보내줄 때 어플리케이션 자체에서 푸시 알람 자체의 설정을 On/Off 해주기만 하면 푸시 알람을 받을 수 있고 블락할 수 있다고 생각했었다. 하지만 지금은 어플리케이션 내부에서 푸시 알람을 설정해주는 것은 어디까지나 SharedPreference와 같은 어플리케이션 내부 저장소에 설정을 저장하게 하는 인터페이스를 이용하는 것에 지나지 않는다는 것을 알게 되었다. 어플리케이션에 대한 푸시 알람의 근본적인 설정은 다른 데 있다는 것? 아무튼…

일단 해보기

NotificationManager (NotificationManagerCompat) 을 써서, 현재 어플리케이션의 알람 설정의 상태를 가져올 수 있다. 잘됬네 잘됬어.

NotificationManager

백그라운드에서 어떤 이벤트가 일어날 때, 그 이벤트를 담당하는 유저가 무엇인지 알려주는 클래스. 즉, 어플리케이션을 사용하는 유저에게 백그라운드에서 어플리케이션에 관련된 무언가를 알려주고자 할 때 사용된다.

Notification 은 다음과 같은 형태를 지닐 수 있다고 한다.

  1. 일반적인 형태의 알람
  2. 하드웨어 디바이스의 특징을 이용한 것 (LED 깜빡임 또는 화면 켜짐)
  3. 진동, 사운드 등으로 알려주는 형태

그리고 이 클래스에 대한 인스턴스를 얻기 위해서는 반드시 다음과 같이 해야한다고 한다.

근데 문제점은 위의 코드에서는 NotificaitonManager 을 쓰지 않고, NotificationManagerCompat 을 쓴 것이 함정. 따라서…

NotificationManagerCompat

옛날 버전의 안드로이드 플랫폼 단말에서도 잘 동작할 수 있도록 지원해주는 호환 라이브러리이다. 방법이 특이한데, 이렇게 써서 인스턴스를 얻도록 해야한다…

from(Context) 메소드는 정적으로 상위 타입이 Context 인 모든 것을 다 받지만, 개인적으로는 앱 자체에 관한 알람 설정을 건드려야 하기 때문에 Activity 가 아닌, Application 혹은 Context 자체를 인자로 넣어주는게 타당하지 않나 싶다.

그리고 나서 다음과 같은 메소드를 통해 현재 Context 에서 Notification 의 설정 여부를 알 수 있다.

Notification

위의 NotificationManager 등을 통해서 유저에게 전송할 수 있는 notification 이벤트라고 한다. 이 때 Notification 을 간편하게 만들기 위한 Notification.Builder(Context context) 가 정의되어 있으니 나중에 필요하면 이걸 쓸 것. 또는 NotificationCompat.Builder(Context context)을 써서 하위 호환성을 맞출 수도 있다.

이 때, Notification 객체는 다음을 반드시 포함해야 한다고 한다.

  1. setSmallIcon(int icon) :: 알람 시 보여지는 작은 아이콘. 인자로는 drawable resource의 id가 와야한다.
  2. setContentTitle(String) :: 설정한 제목
  3. setContentText(String) :: 설정한 세부 텍스트

레퍼런스 사이트에 따르면 다음과 같이 간단한 알람을 생성할 수 있다.

이 때, 하나의 알림은 여러 개의 작업을 트리거할 수 있으며 일반적으로는 어플리케이션의 Acitivity을 여는 작업이 보통이라고 한다. 알림 버튼의 경우에는 Android 4.1 부터 기능을 지원한다고 하니 SDK가 이보다 낮으면 SDK_INT 의 compatibility 확인을 따로 해줘야 할 듯.

나중에 알람쪽에 대해서 한번 자세하게 더 보아야 할 것 같다…
