Zero Initialization

특정 변수에 타입에 맞게 암묵적으로 변환된 zero value 이 세팅되는 것을 말한다.

적용

제로 초기화는 다음의 상황에 따라 여러 가지로 적용될 수 있다.

  • 가장 맨 먼저, const 가 아닌 static 혹은 thread_local (C++11~) 변수의 초기화가 시행된다. 이 때, 이 두 종류의 변수는 다른 값으로 초기화가 일어나기 전에 Zero Initialization 된다. 그 후에 default initializer 가 존재하면 다른 값으로 다시 초기화된다.

  • 클래스가 아닌 타입과 생성자가 없는 클래스 타입들에 대해 적용될 수 있다.

  • 스트링 리터럴에 의해 초기화되는 char[n] 배열의 뒤의 공간에 대해서도 적용될 수 있다.

값

제로 초기화에 의해 초기화된 0 값은 다음과 같이 될 수 있다.

  • T 가 수량 타입(Scalar Type) 이면, 타입 T 에 맞는 0 으로 명시적으로 초기화된다.

  • T 가 유니언이 아닌 타입 이면, 모든 베이스 클래스 혹은 비정적인 멤버는 0 으로 초기화 된다.

  • T 가 배열 타입이라면, 우선적으로 각 요소는 타입에 맞는 0 으로 초기화된다.

  • T 가 reference type 이라면, 아무것도 일어나지 않는다.
