template, auto, decltype (Effective MDC++ No.1..3)

Hexo
2017-11-10
C++

시작하면서

C++98 에서는 타입 추론에 관한 규칙이 템플릿 타입 추론 규칙 하나 밖에 없었다. 하지만 C++11 이후는 auto 와 decltype 및 decltype(auto)(C++14~) 가 추가되면서 여러가지로 형식이 복잡해지게 되었다. 따라서 이 세 가지를 잘 쓰기 위해서는 타입 추론이 어떤 규칙으로 일어나는지 확실하게 이해하지 않으면 효과적인 프로그래밍이 불가능하게 되었다.

타입 추론은 다음과 같은 상황에서 항상 일어난다.

  1. 함수 템플릿 호출 지점에서 발생
  2. 함수 템플릿의 함수 포인터가 사용되는 시점
  3. 표현식에서 auto 가 등장하는 시점
  4. decltype 표현식의 시점 및 decltype(auto) 가 표현식에서 쓰이는 곳

Template 타입 추론

위의 타입 추론이 일어나는 리스트에서 1, 2는 Template 타입 추론을 이용해 컴파일러가 적절한 타입을 써낸다. 함수 템플릿의 선언과 사용은 대체로 이런 모습이다.

2번 예시에서 이 파일을 컴파일 하는 도중에 컴파일러는 expression 을 이용해 두 가지 형식을 연역해야 하는데, 하나는 T의 형식이며 또 하나는 ParamType에 대한 형식이다. 다만 expression은 const 나 volatile와 같은 키워드나 &(lvalue reference qualifier) 또는 &&(rvalue reference qualifier)와 같은 여러가지 예약어가 붙기 때문에 이에 따라 연역을 하는 규칙이 정해져 있다.

예를 들어 컴파일러는 다음과 같이 함수 템플릿의 타입을 추론할 수 있다.

이 경우 x 가 int 이므로 T 또한 int 가 된다. 하지만 ParamType 은 const int& 가 된다. 이와 같이 형인자 T 와 실제 함수 파라미터 ParamType 간의 최종 결정되는 타입이 다를 수도 있다. 함수 템플릿의 타입 추론은 다음 3가지에서 결정된다.

  1. ParamType 이 * 이거나 & 인 경우
  2. ParamType 이 && 인 경우
  3. ParamType 이 Pass by value 인 경우

ParamType 이 * 이거나 & 인 경우

  1. expression 이 & 이거나 && 이면 참조 부분을 무시하고 연역한다. (녹아들어간다)
  2. 그 후에 ParamType 에 대해서 패턴 매칭을 통해 최종으로 T 의 타입을 결정한다.

밑의 예시에서 간단하게 살펴볼 수 있다.

세번째 호출에서는 rx 형식이 const int& 이지만 타입 추론 과정에서 참조성이 무시되므로, T 자체는 비참조로 연역이 되었음을 알아야 한다.

또한, ParamType 에 const 나 volatile 이 추가되면 약간 달라진다.

  1. f(x) 을 호출하면 T == int, param == cosnt int& 가 된다.
  2. f(cx) 을 호출하면 T == int, param == const int& 가 된다.
  3. f(rx) 을 호출하면 T == int, param == const int& 가 된다.

ParamType 이 && 인 경우

이 경우에는 조금 생각해서 타입 추론을 해야한다. 이 경우는 두 가지 세부 케이스로 나뉠 수 있다.

  1. 인수가 오른값이거나 오른값 참조 인 경우
  2. 인수가 왼값 계열인 경우

1번 케이스는 그대로 맨 처음 주 케이스대로 타입 추론을 하면 되지만, 2번 케이스는 약간 생각해야 한다.

  • 만일 expression 이 l-value 계열이면, T 와 ParamType 둘 다 l-value reference 로 연역된다. 따라서 함수 프로토타입을 선언할 때는 && 이지만 실제 연역되는 형식은 & 이다.

ParamType 이 일반 값 (Pass by value) 인 경우

이 때는 함수에 전달되는 인수의 복사본은 전달되는 인수와는 다른 완전한 새로운 객체이기 때문에, 다음과 같은 규칙이 적용될 수 있다.

  1. 이전같이 만일 expression 의 형식이 &, && 이면 참조 부분은 무시된다.
  2. 참조성을 무시한 후에 const 와 volatile 또한 무시한다.
  3. const _T* const 의 타입이 전달될 경우는 const _T* 가 추론이 된다.
