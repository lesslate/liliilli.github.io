Copy elision

C++ 에서, 불필요한 객체의 복사 또는 이동(Copy and Move)를 제거하는 컴파일러 최적화 기법이다. 따라서 리턴 값으로 임시 객체를 만들어 pass-by-value 등으로 전달해야 할 때, 복사 오버헤드를 무효화시킬 수 있다.

다만 이 최적화로 인해 생략된 복사/이동 생성자는 프로그래머의 코드 등에 의해 부작용(side-effects) 을 가질 수 있기 때문에 최적화 시 주의해야 한다. 만약 생성자가 부작용을 가진다면 반드시 최적화를 할 필요는 없다. (다만 최적화 자체는 허용된다.)

예를 들면 다음과 같다.

  • 초기화 시에, 초기화 표현이 prvalue (pure-right value) 이며, 타입의 const, volatile 이 없는 타입이 최종 클래스와 같은 클래스 타입이라면, 다중으로 섞인 초기화 표현은 최종 목적지의 객체에서 한 번만 초기화된다.
  • (C++17) 함수 호출 시에, return value 가 prvalue 이고, 함수의 리턴 타입이 prvalue 의 타입과 같을 때 최적화가 된다. (C++17 부터는 이전 사양에서는 임시 객체(lvalue) 을 반환하는 것과는 다르게 prvalue 로만 반환된다.)
