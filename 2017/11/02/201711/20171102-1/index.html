operator= 자기대입 검증법 대신 쓸 수 있는 방법 (Effective C++ no.12)

Hexo
2017-11-02
C++, Effective_C++

위 방식은 자기 참조 문제에 대해서 안전하지 않게 구현되었다. 즉 rhs 가 자신 객체라면 함수 바디에서 p_bitmap 의 메모리 할당을 해제할 시, rhs 또한 p_bitmap 의 메모리 할당이 해제되어 new Bitmap(nullptr); 과 같은 런타임 에러가 발생되게 된다.

이런 문제에 대해서 대개 이런 방식으로 자기 참조를 막는 코딩을 했고 나도 이렇게 하는게 좋다고 생각을 했었다.

하지만 이렇게 하면 자기 참조 문제는 해결되나 아직 예외에 대해서는 안전하지 못하다. new Bitmap() 에서 예외가 반환 될 경우에는 위의 코드에서는 p_bitmap 은 nullptr 인 채로 자신의 참조형 객체가 반환되게 된다.

그런데 이 방법을 취하지 않고, 명시적으로 자기 참조에 대한 검증을 거치지도 않으면서 예외를 잡을 수 있는 방법이 있었다.

이렇게 하면 효율적으로 자기 검증 뿐만이 아니라, 예외를 처리할 수 있게 된다.
