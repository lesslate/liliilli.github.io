C++11 Multithread Tutorial Series

시험 기간 중이라서 분명 전공 공부를 해야하는데 이런거나 보고 있다. Java 의 경우에는 Thread 을 만들어서 Runnable() 등으로 인터페이스 콜백을 넣는 방법으로 보통의 Thread 을 만든다고 알고 있다. (인자 전달 등은 자세히 해보질 않아서 여기까지만 알지만) C++11 의 경우는 lambda 식 및 functor 그 외 등등을 집어넣어서 함수를 돌리기도 하고, 여러가지 방법으로 안전하게 스레드 간의 통신을 할 수 있게 되어 꽤 편하게 Thread 을 만들 수 있게 되었다. 분명 작년에 OS 과목을 들었을 때 UNIX 환경에서 C 로 Thread 을 짠 기억 때문에 맨 처음 부분에 들어갈 때는 내심 쫄았지만 이런 식으로라면 누구라도 쉽게 스레드를 만들어서 여러가지 프로그램에 적용할 수 있겠구나 생각했다. 지금 생각 난 것인데 Thread 을 이용해 웹 소켓을 이용한 간단한 TCP/IP 통신 라이브러리를 한번 만들어 보고 싶다.
