std::nothrow

Hexo
2017-10-30
C++

std::nothrow_t


개요

operator new 또는 operator new[] 에 쓰이는 인자로, 동적 할당이 오류가 일어나도 예외를 던지지 않고 대신 null pointer 을 던지도록 한다. 기본적으로는 new 연산자가 동적 할당에서 메모리 확보에 실패하였을 때, bad_alloc 예외를 던지게 한다. 하지만 std::nothrow 가 인자로 주어지면 대신에 nullptr 을 반환하게 한다.

std::nothrow 이 자체는 구조체 타입 std::nothrow_t 의 객체 값에 불과하다.

덤, 왜 std::nothrow 가 잘 안 쓰이나요?

원본 출처 https://stackoverflow.com/questions/18061781/why-new-stdnothrow-version-is-not-widely-used

  • nothrow 대신에 기본 bad_alloc 을 던지는 new() 을 더 선호하는 이유가 있나요?

예외의 기본 법칙인, 예외를 내부적으로 처리할 수 없다면, 그 예외에 대해 내부에서 확인하지 않도록 하라 를 따르는 것. 일반적인 리턴 값과는 달리 예외는 무시할 수 없기 때문에 만약 nullptr 을 쓴다고 하더라도 이에 대한 예외를 처리할 수 없음.

  • bad_alloc 과 같은 예외를 체크하지 않는 이유?

보통 메모리 할당 에러와 같은 경우에는 내부에서 컨트롤이 불가능하다. 이 경우에는 프로그램을 종료시키는 것이 가장 좋은 방법이며 처리되지 않은 예외에 대해서도 기본적으로 이 방법을 택한다. 따라서 new(std::nothrow) 가 아닌, new() 을 쓰는 것임.

  • malloc() 과 new 의 차이?

malloc() 은 메모리가 할당되었는가를 체크하기 위해서 부수적으로 NULL 을 체크해야 함. 하지만 new 는 에러 처리 면에서는 보다 추상화되어 따로 에러 체크를 하지 않아도 됨.
