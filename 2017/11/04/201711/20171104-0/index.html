type-traits

컴파일 타임에 타입의 종류를 구분해줄 수 있다. 예를 들면, 특정 타입을 상속받고 있는 타입이거나 특정 타입으로 변환이 가능해야 하거나 하는 정적 분기를 이를 통해 구현할 수 있다.
라이브러리에 포함된 클래스 템플릿은, 대개 메타 펑션 (Meta Function) 이라고도 불린다.

이름 설명
is_void 해당 형이 void 형인가
is_null_pointer 형이 nullptr_t 형인가
is_integral 해당 형이 정수형 (unsigned, int, char…) 인가
is_same 해당 형이 같은 타입인가
is_base_of 해당 형이 어느 타입의 자식 클래스인가
is_convertible 해당 형이 다른 타입으로 캐스팅 가능한가
is_pointer 해당 형이 포인터 타입인가

이 외에도 많은 클래스 템플릿이 존재한다.

위의 코드에서 자세히 살펴보면, false_type 과 true_type 이 각 템플릿 클래스의 기본 클래스로서 정의되어 있는데, 이들은 다음과 같이 정의되고 있다.

true_type 과 false_type 은 실질적으로 integral_constant<bool, _Ty> 의 별칭(alise) 이며 거슬러 올라가면 컴파일 타임 시점에서 value 로서 _Val (true 또는 false) 을 가짐을 알 수 있다. true_type::value 을 호출하면 true 을 얻고, false_type::value 을 호출하면 false 을 얻게 되는 것이다. 이것을 통해 위의 코드에서 템플릿 특수화를 이용해 true 인가 false 인가를 컴파일 타임 시점에서 반환하게 할 수 있다.

또한 integral_constant<class _Ty, _Ty val>은 내부의 클래스와 그 클래스로부터의 기본 클래스로써 많이 쓰여진다.
