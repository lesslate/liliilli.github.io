enable_if 와 SFINAE 훑기

Hexo
2017-11-05
C++

enable_if

컴파일 시에, 조건식이 true인 경우에만 해당 타입이 type 으로서 유효하게 정의되는 타입이다. 만약 false인 경우에는 다른 타입으로 대체할 수 있는 오버로딩 된 버전이 없으면 최종적으로 컴파일 에러을 띄운다.

SFINAE (Substitution Failure Is Not An Error)

오버로딩된 템플릿이 여러 개 존재할 때, 템플릿 매칭 과정 중에 어떤 타입을 대입해서 생성된 코드가 오류가 나더라도 문제없이 매칭되는 다른 템플릿 오버로딩 버전이 있는 경우에는 최종적으로 에러로 취급하지 않는다.

따라서 템플릿을 이용한 여러 개의 오버로딩 된 함수, 등이 있을 때 enable_if을 이용하면 오버로딩과 관련된 타입의 type_traits에 기반해서 (_Test 에 표현식을 써넣는다.) 특정 오버로딩 버전을 선택적으로 배제할 수 있게 된다.

enable_if는 C++만의 SFINAE 을 본격적으로 이용하며 bool _Test 에서 표현식이 true 일 경우에만 중첩 타입이 되어 ::type을 이용해 접근할 수 있게 된다.
