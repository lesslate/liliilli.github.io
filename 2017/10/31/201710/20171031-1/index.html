const 의 사용법 (Effective C++ no.3)

Hexo
2017-10-31
C++

Const

const 키워드가 표의 왼쪽에 있으면 포인터가 가리키는 데이터가 불변이 되며, 표의 오른쪽에 있으면 포인터 자체가 상수이며 데이터의 가변은 가능하나 포인터 자체가 재할당 될 수 없다. 만약 양방향에 const가 존재하면, 포인터 그 자체 뿐만이 아니라 포인터가 가리키는 데이터 또한 불변성을 가진다.

또한, 타입 앞에 const 가 오든 뒤에 const 가 오든 그 의미는 같다.


반복자

STL 반복자는 포인터를 본 뜬 것이며 기본적인 동작 원리가 T* 와 비슷하다. 따라서 어떤 반복자 (iterator) 가 다음과 같다고 할 때,

이 반복자를 무턱대고 const 로 선언해서 써버리게 되면 반복자가 내부에서 가리키고 있는 어떤 배열의 포인터 위치가 불변이 되버려 의도한 동작을 할 수 없게 된다.

따라서 위와 같이 const T 를 받는 반복자를 받고 싶으면 const 을 앞에 붙이는 대신, const_iterator 을 쓰면 된다.

함수 반환 값을 상수로 정해주면, 안정성 및 효율을 유지하면서 터무니 없는 연산을 컴파일 타임에서 막을 수 있다.


상수 멤버 함수 (Constant Member Function)

멤버 함수의 뒤에 붙는 const 는, 해당 멤버 함수가 상수 객체에 대해 호출될 함수임을 알려준다. 이런 함수가 중요한 이유는

  • 클래스의 인터페이스를 이해하기 좋게 하기 위함

  • const 키워드를 통해 상수 객체를 사용할 수 있도록 함.

const 키워드의 유무에 따라서도 멤버 함수들은 오버로딩이 가능하다.

이 때 주의할 부분은 operator[] 의 반환형이 항상 뭔가의 참조형 (&) 을 반환한다는 것이며 참조형이 없으면 안된다. 만약 일반 char 형으로 반환하게 된다면 다음의 코드가 컴파일이 되지 않을 것이다.

기본 타입을 반환하는 함수의 반환 값을 수정하는 일은 절대로 있을 수 없다. 또한 그 뒤에는 반환 시에 ‘값에 의한 반환’을 수행하기 때문에 복제된 반환값을 수정하는 것이 불가능하다. 즉, 수정되는 값은 tb.text[0] 자체가 아니라 tb.text[0] 의 복제본이다.


멤버 함수가 const 라는 것의 의미

bitwise constness

객체의 어떤 멤버 함수가 그 자신 객체의 어떤 데이터 멤버도 건드리지 않아야 const 가 됨을 인정하는 개념. static 등의 정적 멤버는 제외한다.

즉, 그 객체를 구성하는 비트 (bit) 들 중, 어떤 것도 바뀌면 안된다는 성질이다. 따라서 이 비트수준의 상수성을 사용할 경우 ( 대입 등의 ) const 인가 아닌가를 확인할 수 있다.

하지만 이 비트수준 상수성에 대해서는 헛점이 존재한다.

ctb 자체는 상수 객체임에도 불구하고 이런 꼼수를 통해서 상수 객체의 비트를 흩뜨릴 수 있게 된다. 따라서 이런 상황을 방지하기 위해 밑의 개념이 나오게 되었다.

logical constness

논리적 상수성이란 개념은 한 비트도 바꿀 수 없다고 제창되는 비트 상수성에서 좀 더 느슨해진 개념이다. 상수 멤버 함수라고 해서 객체의 한 비트조차 바꿀 수 없는 것이 아니라, 일부 몇 비트 정도는 바꿀 수 있되 그것을 사용자 측에서 알아채지 못하게만 하면 상수 멤버 자격이 있다는 것이다.

따라서 이 논리적 상수성을 만족 시키기 위해서 기존의 객체를 다음과 같이 만들 수 있다.

mutable 은 외부에서는 영향을 주지 않는 (사이드 이펙트가 없는) 타입 한정자이지만, const 와는 다르게 내부에서는 값을 변경할 수 있어 일종의 논리적 상수성을 유지할 수 있게 해준다.


상수 멤버 및 비상수 멤버가 오버로딩 시, 코드 중복 현상을 피하는 법

만약에 위의 짧디 짧은 코드가 매우 길어지게 된다면, 코드 중복과 함께 컴파일 및 유지 보수 등이 매우 힘들어진다.

이를 막기 위해서는 별도의 멤버 함수를 만들어서 옮겨두고 호출하게도 할 수 있지만 컴파일러가 보기에는 중복된 코드를 잠시 추상화시킨 것에 불과하다.

따라서 가변 멤버 함수와 불변 멤버 함수가 오버로딩으로 존재하면, 코드 중복성을 줄이기 위해 가변 멤버 함수에서 불변 멤버 함수를 불러와 캐스팅하여 반환하도록 하는 것이 좋다.
다만 주의해야 할 점은 상수 멤버 함수에서 일반 가변 멤버 함수를 불러서 캐스팅 후 반환을 하는 것은 금지된다. 상수 멤버 함수에서는 자기 자신을 포함해서 비트가 변경될 여지를 남기지 말아야 하는데, 상수 멤버에서 일반 함수를 불러오기 위해 const_cast<TextBlock> 을 실행하여 자기 자신을 변경할 여지를 남기기 때문이다.
