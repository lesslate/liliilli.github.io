std::vector<bool>

Hexo
2017-11-10
C++

std::vector

기존의 std::vector<_Ty> 이라는 템플릿 클래스가 있어, 다른 형태는 이를 써도 되지만 bool 형태는 공간을 최적화 하기 위해 비트셋을 이용하는 방법으로 약간 특수화되서 제공된다. 따라서 bool값을 저장할 때 한 원소를 저장하는데 sizeof(bool) 을 쓰지 않고 대신에 단일 비트을 써서 저장한다.

만약 저장할 boolean 배열의 길이를 컴파일 타임에 알 수 있다면, std::bitset 을 사용하는 것이 정신 건강에 좋다.

사용 시 주의점

  1. 연속 배열와 같은 방법으로 값을 저장해서는 안된다.

  2. std::vector<bool>::reference 을 vector 컨테이너의 접근자로 지정해줘야 한다. 대표적인 예시로 operator[] 연산자의 오버로딩에서 값 반환으로 ::reference 을 반환한다.

  3. 비트 값을 만들 때, std::allocator_traits::construct을 쓰지 말아야 한다.

  4. 멀티 스레드 환경에서, 각기 다른 스레드가 같은 컨테이너의 다른 원소를 변경하지 말도록 해야한다.

  5. 컨테이너 내부의 원소를 받을 때 auto 을 이용해서는 안된다. 명시적으로 bool을 표기해 줘야 한다.

주의점 5번 심화 탐구

2번에서도 알 수 있듯이, vector<bool> 의 operator[] 은 _Ty 을 반환하는게 아니라 std::vector<bool>::reference 을 값으로 반환한다. 이 reference 객체는 접근을 용이하게 하기 위해 사용되는 nested class 이며 단일 비트의 행동을 따라하는 프록시다.

느낀 것.

  • std::bitset 에 대해서 한번 공부해보고 싶다.
  • iterator 의 종류와 세부 사항 및 요구 조건에 대해서도 한버 알아보고 싶다.
  • 도중에 자료를 찾다가 느낀건데.. 연속 요소 저장 컨테이너인 vector 의 resizing 에 대해 재밌는 글이 눈에 띄어 한번 읽어보고 정리를 해보고 싶다.
